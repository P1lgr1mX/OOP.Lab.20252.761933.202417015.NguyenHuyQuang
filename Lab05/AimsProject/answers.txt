1. Comparison between Swing and AWT top-level containers

In Java GUI programming, top-level containers serve as the essential root windows that interact directly with the operating system to host other user interface elements.

    AWT Containers: The original framework uses foundational heavyweight windows, primarily including Frame (a standard window with a title and borders), Dialog (a secondary pop-up window), and Applet (for browser-embedded apps).

    Swing Containers: Swing provides enhanced equivalents to manage modern windows, named with a 'J' prefix: JFrame, JDialog, and JApplet.

Architecture and Component Hosting:
While Swing's top-level containers themselves remain heavyweight (managed by the OS), the child components inside them are lightweight (rendered purely in Java). Because of this architectural difference, you cannot add lightweight components directly to a Swing container. Instead, they must be explicitly added to the container's content-pane (via getContentPane()), which manages the layout and proper rendering of the UI.
2. Comparison between AWT and Swing component names

The transition from AWT to Swing introduced a highly structured naming convention. This helps developers easily distinguish between the older, platform-dependent components and the newer, cross-platform Swing elements.

    The Naming Rule: Swing component classes typically start with a 'J' prefix, followed directly by the traditional AWT component name. This indicates that Swing components inherit from javax.swing.JComponent rather than the native java.awt.Component.

Examples of the Mapping:

    Label (AWT) → JLabel (Swing): Used for displaying static text, but Swing's version also supports images and rich formatting.

    Button (AWT) → JButton (Swing): Triggers click events, with Swing adding support for icons and rollover effects.

    TextField (AWT) → JTextField (Swing): Provides a single-line input field with improved text handling capabilities.

    Panel (AWT) → JPanel (Swing): A basic layout container, where Swing's version brings built-in double-buffering for smoother rendering.
